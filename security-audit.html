🔴 CRITICAL — XSS via innerHTML (FIXED)

Previously vulnerable to XSS via unescaped API data in innerHTML assignments.

Fix: Added esc() function to sanitize all API-derived strings before HTML injection.

function esc(str) {
  const d = document.createElement('div');
  d.textContent = String(str || '');
  return d.innerHTML;
}

🔴 HIGH — Content Security Policy (HARDENED)

Removed 'unsafe-inline' from script-src by moving JS to external file.

🔴 HIGH — API Key Exposure

Etherscan API key is stored in client-side JS. If set, it's exposed to users.

Recommendation: Use server-side proxy for API calls.

🔴 HIGH — Subresource Integrity (MISSING)

No SRI hashes for external resources.

Recommendation: Add integrity attributes to font links.

Other Fixes

  • Added HSTS header for HTTPS enforcement.
  • Implemented client-side rate limiting (10 scans/min).
  • Added basic API response validation.
